# RAG 入门指南

## 什么是 RAG

RAG（Retrieval-Augmented Generation，检索增强生成）是一种将信息检索与文本生成相结合的技术架构。它的核心思想是：在 LLM 生成回答之前，先从外部知识库中检索相关信息，然后将检索结果作为上下文提供给 LLM，从而减少幻觉、提升回答的准确性。

## RAG 的核心流程

RAG 系统通常包含两个阶段：

1. **离线索引阶段**：将文档加载、切分、向量化后存入向量数据库。
2. **在线检索阶段**：接收用户查询，从向量数据库中检索相关文档片段，拼装成上下文，交给 LLM 生成回答。

## 为什么需要 RAG

- **解决幻觉问题**：LLM 可能会编造不存在的事实。RAG 通过提供真实的参考资料来约束 LLM 的输出。
- **知识更新**：LLM 的训练数据有截止日期。RAG 可以检索最新的文档，保持知识的时效性。
- **领域适配**：通用 LLM 不熟悉特定领域的知识。RAG 可以让 LLM 基于企业内部的文档回答问题。

## 常用的 Embedding 模型

中文场景推荐使用 BGE 系列模型，如 BAAI/bge-large-zh-v1.5。它由智源研究院训练，在中文语义理解任务上表现优异。
